# 2026-06-09 — Cross-Device Web UI Issue

## Problem
User testing OpenClaw Web UI from Windows 11 + Edge browser. Voice, Attach, and settings gear buttons were unresponsive/grayed out.

## Root Cause
Browser **secure context** requirement. Modern browsers (Edge, Chrome) block certain APIs when accessing OpenClaw via IP address over HTTP:
- Microphone/Voice API — requires HTTPS or localhost
- File picker/Attach — restricted on insecure origins
- Device identity — blocked

Works fine on `http://127.0.0.1:18789` (localhost counts as secure context).
Fails when accessing via `http://<ip-address>:18789` from another device.

## Solutions

### Option 1: Localhost (same machine)
Access via `http://127.0.0.1:18789` — all buttons work.

### Option 2: Tailscale Serve (recommended for remote)
```bash
tailscale serve --https 443 --set-path 18789 http://127.0.0.1:18789
```
Then access via `https://<machine>.<tailnet>.ts.net`
- Automatic HTTPS certs
- All browser APIs functional
- Secure device identity

### Option 3: Control UI insecure auth (partial)
Add to `~/.openclaw/openclaw.json`:
```json5
{
  gateway: {
    controlUi: { allowInsecureAuth: true }
  }
}
```
⚠️ Only fixes Control UI auth — Voice/Attach still blocked by browser on non-localhost HTTP.

## User Setup
- OpenClaw gateway running on Mac Studio
- Testing from Windows 11 + Edge (failed — HTTP over IP)
- Has Tailscale on phone, considering full Tailscale setup

## Reference
- Browser secure context docs: https://developer.mozilla.org/en-US/docs/Web/Security/Secure_Contexts
- OpenClaw gateway config: `gateway.controlUi.allowInsecureAuth`

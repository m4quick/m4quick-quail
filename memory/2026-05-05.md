## OpenClaw WebUI Troubleshooting - Greyed-out Buttons

**Issue:** Buttons in OpenClaw WebUI are greyed out and unresponsive

**Testing History:**
- **Mac + Safari:** Buttons greyed out, non-functional
- **Windows 11 + Edge:** Same behavior - buttons greyed out, non-functional  
- **Mac + http://127.0.0.1:5001:** Same behavior - no change
- **Result:** Issue is reproducible across browsers/platforms - not browser-specific

**Root Cause Hypothesis:** JavaScript not loading properly (icons appear greyed out as if JS isn't initializing)

**User Feedback:** Session continuity between devices needs improvement - same user sessions should share context

**Root Cause Identified:** HTTPS requirement - WebUI likely needs secure context (HTTPS or localhost) for full JavaScript functionality

**Next Steps:**
- Try https://localhost:5001 instead of http://127.0.0.1:5001
- Check if OpenClay gateway supports HTTPS on that port
- Verify SSL/TLS certificate is configured
- Browser DevTools (F12) → Console to confirm "insecure context" warnings

**Session Continuity Issue:** WebUI sessions don't share history across devices. Documenting key findings in this file for persistence.
